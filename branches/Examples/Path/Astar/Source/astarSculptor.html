AStar Pathfinder Landscape Sculptor

  • General Use
    • Cloned from Mattias Landscaper with changes and addittions.
      There are four (4) pages: Terrain, Slope, Texture, and Terrain Attributes.
      • Terrain: Create .. Edit .. Save the Heightfield
        • Select size
        • Press GO
        • Edit using the Mouse and the Trackbars at the bottom of the form to adjust the size and depth: darkness.. elevation change
        • Save
      • Slope: Create .. Save the Slope
        • Select size
        • Press [Slopes]
        • Press [Save Slope.aef]
        • Make a Slope at 0.01 and save it. change to 0.3 and create another [Slopes] to use when the Texture is created.
          Using the Slope created at 0.01 will make the texture have too many slopes.. it is used to have a Range of Slopes. The 0.3 Setting only has the Steep slopes.
          As another option the size cna be changed to blank out the slope prior to Texture creation.
      • Texture: Create .. Save the Texture
        • Select size
        • [Create]
        • [Save..]
        • Paint more detail before or after using the Terrain Painter
      • Terrain Attributes: Create .. Save the Attributes
        • Select sizes and Load the Texture [Background]
        • Click different attributes to change Colors and their attribute values
        • Click .. Drag.. Paint your Scenario
          • Use the Big brush to quickly paint, then convert to Map Attributes, and Repaint.
        • Save Map Att .amf
        • Save G Cost .agf
        • Save MAG B .abf
    • Requires these 256x256 textures be in the Textures directory: ssand.jpg, vgrass.jpg, vforest.jpg, srock.jpg, hsnow.jpg, srocky.jpg (Replace as desired.. rename or recompile)
    • A Project Terrain Data File (adf) requires a heightfield.bmp, (Map Attributes (Combined) File (amf): thus a slope.aef is suggested), a Texture.bmp, MAG Values B .abf, and a G-Cost .agf

Description of all the buttons and baubles

  • Terrain
    • [Landscape Size] : 256, 512, 1024
    • [Load]
    • [Save]
    • [Clear]
    • [GO]
    • Memo area
      • Using cup \/ or bubble /\ as second line changes the 'form' of the heightfield
    • [Reset Default Memo] if changed into oblivion
    • [w/Island] creates a lowered edge
    • [Standard] redisplays
    • [Render water] colors in the lower areas
    • [Rescale] might be meaningless without a data input area.
    • [Subdivide] ... ?
    • [Height altering] is used to 'Edit' the heightfield
      • [Radius] slider changes the width of the 'paintbrush'
        • [Box] if checked the paintbrush is square, unchecked it is round.
      • [Depth] slider changes the density..darkness of the 'paintbrush'
  • Slope
    • [Slope Size] is normally 2, 4, or 8 times the size of the Terrain heightfield
    • [Slopes] renders the slopes iaw
      • the density data input: [0.01]
    • [Save bitmap] saves the slope image... it can be used as the Texture background when painting the Terrain attributes
    • [Save Slope.aef] saves the .aef and a .bmp, colored iaw the slope ranges, also can be used as the Texture background or Twix image when painting the Terrain attributes.
      • Panels and Data entry areas enable changing the Colors (Click the Panels) and the Slope Attribute values. (This is the only place to change them)
    • [Slope or Texture] toggles which image to render to:
      • [Shaded] will darken the image
      • [AA Basic] Anti Alias
      • [AASweep] " " ...
      • [Shadow] more darkness
  • Texture
    • [Texture Size] is normally 2,4,8 times the Heightmap size
    • [Create] the Texture iaw the parameters
    • [Shaded] will do all the Slope Shady stuff to the Texture.
    • [Ambient] data input is another thing to decipher
    • [Save...] the Texture to use with the heightfield and as the background during Terrain attribute painting.
    • [Flip n Save...] to use as a Scenario Planner. Be advised what you see here will be flipped vertically when viewed in 3D.
    • The Sand, Grass, Forest, Rock, Snow, and Slope Modifiers determine the range of coverage of the texture image; or the color, if no image is found.
      • Button to choose the texture image
      • Button to choose the color
      • 4 data edit areas (Sand used as example). StartHeight, FullOnHeight, StopHeight, FullOffHeight
      • [-2] : StartHeight
      • [-1] : FullOnHeight
      • [0.2] : StopHeight
      • [0.3] : FullOffHeight
    • [Paint Grid] then [Flip n Save...] to use as a Scenario Planner.
      • The Panel can be clicked to select the Grid Color
      • Grid Size selects from 64x64, 32x32, 20x20, 16x16, 10x10, 8x8
  • Terrain Attributes
    • [Image Size] Ditto on 2, 3, 8 multiple of height.
    • [Tile Size] determines the density of the pathfinding (Image 2048 at Tile 1x1 takes ~300meg, so 4096 takes 4 times that... 1.2 Terabyte)
    • [New] blanks out and sets up
    • [Base] creates the texture using only colors.. in the Background image. But they are lerped so are no longer the same as the Attribute colors.. still under construction... a Terrain Attributes autofill
    • [Background] loads an image to use as the background... paint by numbers.
    • [Save Back...] as the [Paint Map] toggles painting the background or the Terrain Map Attributes... the background might become art. Use to Mark Roads, Lakes, ... to Paint in the Terrain Painter or use as a locator for 3D Objects.
    • [Load Map...] enables finishing later.. just be sure to have the Big Brush toggled, then Convert and Repaint.. as it is Only an image when loaded. Can also be used to have several versions of the Attributes.. Change the values (but not the colors), Load the image then save the Attribtes as a different file name.
    • [Save Map...] refrigerator art.. see above.
    • [Paint Map] toggles painting the background or the Terrain Map Attributes.
    • the [slider] controls the transparency of the Attributes.. to check out the background to atttribute consistency.
    • There are Big Brush, File saving and Map Attribute areas
      • Big Brush is used to Rapidly paint Map Attributes.. BUT the [Convert] button MUST be pressed after painting to convert 'colors' into actual attributes and the related G-Costs.
        • [_] checkbox toggles painting with the Big Brush
        • Radius [Slider] controls the size of the paitbrush
        • [Convert] Big Brush paint into 'stored' Map Attributes and G-Costs arrays.
        • [RePaint] using Map Attributes at the designated tile size (image may become 'blocky'.. as it is now tiled.
      • [Twix] loads an image to use as an intermediate comparison layer. The Slider controls its transparency. Use to compare the Texture to ____ (Slope) to Map Attributes.
      • [Units] loads Unit Locations to use as an intermediate comparison layer. The Slider controls its transparency.
        Use to compare the Locations to ____ (Slope) to Map Attributes.
        The other Slider controls the Size, so even at 1x1 they can be larger, thus Visible.
