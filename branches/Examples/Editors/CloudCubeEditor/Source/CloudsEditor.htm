GLScene Gls Cloud Cube Editor

  • The purpose is to create clouds using Cube holders of sprites or particles. The area to the right contains controls to do this. At its bottom are buttons to activate other forms to make the textures. The top portion controls the current Cube and the Cubes page(s) contain parameters OF the sprites IN the Cube, the Global page contains Global parameters.

  • Cloud Image Texture hints
    • Textures are thought to be 128x128, there might be code Expecting that size.
    • Make sure Textures are NOT Full frame (they should have Black edges) else Square edges will appear. Edit if necessary using the TextureEditor. Textures can be made smaller or larger than 128 then resized, pasted... to MAKE the image.
    • Makers: The image can be saved used the Save Image button. Changing the Image size to larger than 128 (256 or 512) may allow cropping the image to select a portion for use. Numbering the images (for example: a1..a16 or b1..b16) will aid selection into the texture combiner and keeping the images organized.

Gls Cloud Cube Editor How To

  • Install
    • Make a Clouds directory and unzip the file in it.
    • Make an Images directory
    • Run the program and create..save..move cloud images into the Images directory. Use the Makers to create cloud textures. MF Clouds is easiest, the others create 'special' clouds.
  • Make a Cloud
    • No matter the Cloud type , AT LEAST 1 file must be selected and the Button on the All page pressed.
      • First the Texture images must be selected (for all the clouds). See Cubes Image help ... Click a button to load the file name, Click the box to Toggle using That texture, and set the Max percent the cloud will use that texture. The Max of all images used Must be > 100. If necessary set the last image to 100. The images will be used, 1 per texture (if they are set to be used for that cube), repeatedly until they have been used their max amount. The Min value has not been utilized, other than it must be > 0 for the texture to be used.
      • Set the Alpha and Type for all images (or ignore it, the default works for me, kinda, they do seem to have dark edges..rather than going light to transparent) (Type is ignored, as nothing uses other than 1 image per file, yet. When it is implemented, only PFX sprite types will use more than 1 image per texture). The File name, Alpha and Type are actually Global.. for all the Cubes to be made. The Use toggle and the Percent are set Per Cube.
    • Set the Cubes Size parameters: number of sprites, X,Z range, etc...
    • Set the values for the Cube on the Clouds page (Position, Size, (do not use Rotation, and speed is currently ignored)). Press the black lettered Add button to create a Cube.
      • Click the Cube (in the center) and move the Cursor. The new Cube should be what is moved. Position it as desired. Yes even after setting the Position the Cloud cube can be re-positioned and Resized. Hold down the [CTRL] key while moving the Gizmo Cursor and the Cube will change size.
      • It helps to move the Cube or sprite a little in the direction desired and then press the [CTRL] key to activate the resizing. Also using the Top left corner, rather than the individual arrows is sometimes easier, especially when a Cube or Sprite becomes very very small.
    • Press the Generate Button to make the Sprites for the Cube. The Regenerate button will regenerate (remake the sprites) ... if the Size range Min Max is not the same they will be different sizes.. and the position will change.
    • The Sprites button box toggles the display of the Cube, Sprites, or both. This helps selecting the Cube or a Sprite to move..Resize. There is a 'bug' in the PFX sprites.. in that they do not 'disappear' like normal sprites... O well. It also takes a bit to make them go away after pressing [New]. Click Off in the Sprite Display and Press Regenerate... they should disappear then.
  • Press the Save button to save the Cloud data into a file. Note that a Cubes Sprites can be Moved..Resized beyond the values set for them, Sprites can be positioned outside of their Cube.. if Regenerated, they will become 'normal' again. Once saved, a Cloud Cube can later be loaded using the Open button.

Gls Cloud Cube Editor Things

  • Top Portion
    • New: Clears anything created, resets to Defaults
    • Open Button
      • File name display is in the form caption (.dat is the file extension)
    • Save Button activates a File name selector
    • Exit Button closes the application.
    • Add a Cube using current parameters (here and in the Cubes pages)
      • Blue Add: Add at Blue Cube position
      • Red Add: At last location
      • Add Open...: Is Complicated. It loads a .glsm or dxf file containing a 'shape' that will be filled with Sprites. To use it the Cubes/Type/ Cloud Pattern must be set to 'Fill Shape...' and the Cubes/Type/ Generation Method set to 'Fill Shape...'. [or not.. they will be set when loading] The checkbox toggles using input Cube sizes (X, Y and Z size on this page) or a Default 1 as size. (The shape is resized from whatever size it was designed as) The other checkbox toggles displaying the freeform (to check that the Shape Up X,Y, or Z is correctly set). The other checkbox toggles setting the Shape Up direction Negative or not. The Edit box with the 6 in it is the Resolution..density of the Sprites... Used as X,Y,Z so 6 become 6x6x6=216 possible sprites... Not all are in the Shape, unless it is just a cube. Those Not in the shape are culled.. Setting the Cubes/Size number of parts (sprites) is ignored... but it is filled in with the number actually used, after generation. Textures (Cubes/Image) must be set prior to generation, or a Cloud.dat file opened and its textures used. The Progress bar between the Generate buttons displays progress during shape file loading (which happens during the first Generation) and Sprite location determination. The checkbox there toggles using input Sprite sizes (Variable X and Z size) or a Default (all are square size and a percentage of the Shape size).
    • The Arrow buttons select the 'Current Cube' and changes the Text display to that cubes data. This can be used to 'Edit' .. Change the current Cubes Parameters. Especially helpfull when a Cube becomes inverted.. just delete the Negative sign on the Size. [Reset] the Cube after changes to 'Set' the new parameters.
    • [Delete] the last cube made.. maybe..might not work. There is no Undo.. Save per Good thing created. There is no Multiple file loading to combine cloud files.
    • [Reset] is used to Reset (change) the Cube parameters to what is set in the text inputs, after changing the Cube Text viewed using the Arrow buttons.
    • Position X,Y,Z: Can be changed by the Mouse using the Gizmo or manually changed and Reset to the new values. This can be usefull as a Cube can get lost.
    • Size of the cube: All sprites will be contained Within this area. Can be changed by the Mouse using the Gizmo or manually changed and Reset to the new values. This can be usefull as a Cube can become negative.. hard to tell.. but if there is a - sign in front of the number.. manually delete the minus sign and press the Reset button.
    • Rotation: uhm leave at 0,0,0 to avoid confusion as the Sprites will not be in that rotated area.
    • Speed is per X,Y,Z direction, not actually implemented yet. Sprites will be moved individually to reduce the lag.
    • Cube Group is used to visually link groups of cloud cubes.. someday will be used to apply additional effects.
      • Color panel
      • Text input
  • Middle
    • [Sprites: On, Off, And] used to view Cubes, Sprites, or Both and allow easier visual editing of one or the other.
      • On: Sprites ON Cubes OFF, Off: Sprites Off Cubes ON, And: Sprites ON Cubes ON
    • Generate sprites for New cubes, keeping Sprite data already generated
    • ReGenerate data for ALL Cubes: Sprites
  • Bottom
    • The Maker and Editor buttons linked at the top of this page.
    • The little Box on the left toggles the display of the Grid, so you can create a Cloud, Save it using [Image Capture], and use IT as a Cloud Texture later.
    • The other little Box on the left toggles the display of the Grid, so you can create a Cloud, Save it using [Image Capture], and use IT as a Cloud Texture later.
    • [Image Capture] Button: Pressing the dots pops-up a dialog to input the file name of the image to be saved. The order of the dots indicates the size of the Image: None (place holder) Current Screen size, x1(32bit), x2(32bit), ~1 Square(32bit)
    • A Help button (this).
    • The little Box on the right toggles the display of FPS information.
    • The other little Box on the right toggles the Use of a Screen refresh (.. can 'pause' PFX Sprite creation-destruction) in the display of FPS information.

Gls Cloud Cube Editor Screen: Viewer

  • Mouse
    • Left Button
      • Move the Selected Object (See ___ for a description of Movement). The Gizmo will display Arrows when an object is selected. The selected objects Name, X,Y,Z location will be displayed in the Status Bar at the bottom of the viewer.
      • While pressing the [CTRL] key: Resize the Selected Object
    • Right Button
      • Rotates around the Target Object (Normally in the center: 0,0,0)
      • While pressing the [CTRL] key: Resets the Target Object to Center 0,0,0
  • Keyboard
    • [Esc] Quits..Exits the program
    • Camera Zoom Keys
      • [Z] Zooms In
      • [C] Zooms Out
    • Target Object Movement
      • Up: [W]
      • Down: [S]
      • Left: [A]
      • Right: [D]
      • Forward: [Q]
      • Reverse: [E]
  • Screen Color is Set in Global Environment
  • Sun Color is Set in Global Environment
  • The Status bar at the bottom displays the Selected object, X,Y,Z positions, and in the far right display box: FPS: Frames Per Second and the number of Sprites, Particles, PFX CS Particles, PFX Fog, PFX PL Mask and the Sort time (of the PFX particles)
